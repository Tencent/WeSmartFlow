# 知识卡片生成

用 `generate_html_card` 生成单页 HTML 知识卡片。

## 核心原则

快速学习首轮固定生成卡片组；首轮之后，卡片不是默认动作，而是「值得沉淀复习」时的动作。右侧辅导聊天栏需要先判断用户语境：

- 第一次有效交流 → 必须自主判断需要几页，并生成 3-5 张图文并茂的入门卡片组。
- 用户想交流、追问、澄清、确认理解 → 直接对话，不生成卡片。
- 用户明确要求卡片/总结/图解/可视化，或正在系统讲解新的稳定概念 → 生成卡片。
- 若预计文字回复会超过 200 字，或需要公式、推导、动画、表格、图片、插图、流程图等展示 → 生成卡片。
- 刚生成过相关卡片后的跟进问题 → 优先对话解释，不重复生成。

## 生成判断

问自己三个问题：

1. 这次内容是否是稳定知识点，而不是临时闲聊/简单追问？
2. 用户以后是否会想回头看这份材料？
3. 当前上下文是否还没有类似卡片，或者用户明确要求新卡片？

三者大多为「是」才生成卡片；否则用对话交流。

## 首轮页数判断

- 简单主题：3 张（概念、机制、例子/应用）
- 常规主题：4 张（概念、结构/流程、关键细节、应用/误区）
- 复杂主题：5 张（总览、子概念、流程/对比、应用、总结）

首轮至少 1-2 张卡片应明确要求插图、流程图、对比表或公式可视化，让卡片图文并茂。生活化、地理、历史、文化、结构/空间关系、抽象类比等主题优先使用真实插图，不要过度回避图片生成。

## 调用方式

每张卡片只承载一个概念：

```
generate_html_card(
  title="卡片标题",
  content="详细描述卡片内容：要展示哪些要点、用什么形式（表格/公式/流程图/对比等）",
  node_ids=["节点ID"]  // 可选，关联知识图谱节点
)
```

`content` 越具体，卡片质量越高。要说明展示结构，但不要把多个子概念堆进一张卡片。
